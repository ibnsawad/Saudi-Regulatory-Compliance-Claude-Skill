# NCA DCC-1:2022 — Data Cybersecurity Controls

## At a glance

| Field | Value |
|-------|-------|
| Issuer | National Cybersecurity Authority (NCA), KSA |
| Document | Data Cybersecurity Controls (DCC-1:2022) |
| Status | Mandatory for in-scope entities |
| Relationship to ECC | DCC overlays ECC and DCC; ECC remains baseline |
| Relationship to PDPL | DCC is cybersecurity-of-data; PDPL is privacy-of-personal-data — both apply to personal data |

## Purpose

Provides cybersecurity controls focused on protecting data through its full lifecycle — creation, processing, storage, transfer, sharing, disposal — irrespective of the system or environment that holds it.

## Applicability

Applies to all entities in NCA scope. Within those entities, applies to:

- Government data (with overlays from NDMO standards)
- Sensitive business data
- Personal data (alongside PDPL)
- Data classified at or above the entity's confidential threshold

## Data classification (KSA government scheme)

KSA uses a four-tier classification under NDMO Standards, mirrored in DCC implementation:

1. **Top Secret** (سري للغاية)
2. **Secret** (سري)
3. **Confidential** (محدود / مقيد)
4. **Public** (عام)

For private-sector entities, DCC requires the entity to define and operate a classification scheme that is at least as protective. Most map their internal scheme back to the NDMO four levels for government engagements.

## Domain structure

DCC-1:2022 uses a domain structure parallel to ECC's, focused on data:

1. **Data Cybersecurity Governance** — strategy, policy, roles (data owner, data custodian, data steward), classification scheme, lifecycle definition, risk management for data.
2. **Data Cybersecurity Defence** — classification and labelling, access governance, encryption at rest, encryption in transit, key management, masking and tokenisation, redaction, DLP, secure data sharing, secure data disposal, data integrity, secure backup, anti-leakage.
3. **Data Cybersecurity Resilience** — backup integrity and recovery testing, ransomware-resilient storage, immutable copies for top-tier data.
4. **Third-Party and Cloud Data Cybersecurity** — sharing with third parties, data-processing agreements, cross-border transfer, cloud-resident data controls.

## Lifecycle expectations

A robust DCC programme covers each stage:

- **Create** — classification at creation; data-owner assignment.
- **Store** — encryption commensurate with classification; access governance; backup with integrity.
- **Process** — least privilege; logging of access to sensitive data; redaction in non-prod.
- **Transmit** — encrypted channels; integrity controls; controlled gateways; secure email and secure file transfer for restricted data.
- **Share** — DPA / data-sharing agreement; recipient assurance; logging; expiration.
- **Archive** — long-term retention with integrity controls and key continuity.
- **Dispose** — secure deletion; cryptographic erasure for cloud; certificates of destruction for media.

## Audit hot spots

- No formal classification scheme, or a scheme defined but not operated
- Sensitive data found in non-prod environments without redaction
- Encryption keys co-located with the encrypted data
- DLP deployed but not tuned, or not covering email + endpoint + cloud
- No documented data flows or data inventory
- Cross-border transfers occurring without a transfer mechanism (intersect with PDPL Transfer Regulation)
- Disposal evidenced for media but not for cloud storage and backups
- Backups encrypted but kept where ransomware can reach them (no immutability / offline copy)

## Required artefacts

- Data classification policy and scheme
- Data inventory (records of processing, sources, recipients, retention, location)
- Data flow diagrams for sensitive data
- Encryption and key-management policy with key inventory
- Data-sharing and third-party transfer register
- DLP policy and deployment evidence
- Backup and recovery plan with restore tests
- Disposal procedure and certificates / cryptographic-erase evidence

## Mapping

- ECC-2:2024 § Cybersecurity Defence — Data Protection
- PDPL Articles on lawful processing, security, retention, and breach notification
- NDMO Personal Data Protection Standards (for government entities)
- ISO/IEC 27001:2022 Annex A.5.10 (Information classification), A.5.12, A.5.13, A.8.10, A.8.11, A.8.12 (data masking, leakage prevention), A.8.24 (use of cryptography)
