# NCA TCC-1:2023 — Telework Cybersecurity Controls

## At a glance

| Field | Value |
|-------|-------|
| Issuer | National Cybersecurity Authority (NCA), KSA |
| Document | Telework Cybersecurity Controls (TCC-1:2023) |
| Status | Mandatory for in-scope entities supporting telework |
| Relationship to ECC | TCC overlays ECC for telework scenarios |

## Purpose

Establishes cybersecurity expectations for telework — i.e. employees and contractors performing work-related activities outside the entity's facilities, accessing entity systems and data remotely.

## Applicability

Applies to NCA-scoped entities that allow any of the following:

- Permanent or hybrid remote work
- Field-based access to corporate systems
- Contractor remote access
- BYOD where used for work
- Travel-based access for executives or sales

## Coverage areas

TCC structures controls around the realities of telework:

1. **Telework Governance** — telework policy, defined eligibility, documented authorisation workflow, awareness for teleworkers, periodic review.
2. **Endpoint Security for Telework** — managed device for sensitive access, EDR, hard-drive encryption, screen lock, anti-tamper, baseline hardening.
3. **Remote Access Architecture** — VPN or ZTNA, MFA (phishing-resistant for privileged access), conditional access policies, posture checks, split-tunnel disallowed for sensitive workloads, secure DNS.
4. **Data Protection in Telework** — DLP, restricted local storage, controlled clipboard / file-transfer to BYOD, screen-capture restrictions for sensitive applications, secure print policies.
5. **Identity for Telework** — strong authentication, just-in-time elevation, no shared accounts, session timeouts.
6. **Monitoring and Logging** — capture of remote-access events, anomaly detection, review of geo-impossible logins, blocked-country lists where appropriate.
7. **Awareness and Training** — telework-specific awareness, secure home network basics, phishing reporting.
8. **BYOD** — registration, mobile device management, separation of personal and work data.
9. **Contractor and Third-Party Telework** — contractual controls, time-bound access, dedicated jump infrastructure.

## Audit hot spots

- VPN without MFA, or MFA via SMS only
- Posture checks not enforced on connecting devices
- Personal devices accessing sensitive data without MDM
- Sensitive data downloaded locally and unencrypted
- No geo-fencing or anomaly detection on remote access
- Contractor access not time-bound
- Awareness training generic; not tailored to telework risks

## Required artefacts

- Telework policy with eligibility and approval workflow
- Remote access standard (technical baseline)
- Endpoint baseline for managed devices
- BYOD policy and MDM enrolment evidence
- DLP rule set for telework
- Conditional access / posture policy configuration export
- Telework-specific awareness completion records
